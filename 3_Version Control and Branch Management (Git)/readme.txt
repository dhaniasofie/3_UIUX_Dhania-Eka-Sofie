versioning/version control berfungsi untuk mengatur file c dari source program
masalah yang sering muncul pada revisi penamaannya akan selalu berbeda dan akan memiliki banyak file/nama yang dibuat

Tools:
1. git, salah satu version control system populer yang digunakan para developer
untuk mengembangkan software secara bersama-sama
git repository: 
- folders & files
- history
2. vs code
3. github

git push: kalau sudah dicommit, siap dikirim ke repository >> untuk ditampilkan di
repository

git add: memasukkan semua file perubahan

git commit: untuk message menerjemahkan apa yang sedang dilakukan

diff: untuk mengetahui bagaimana perubahan yang terjadi di file yang telah diedit
stash: untuk menyimpan perubahan sehingga dapat dikembalikan di perubahan-perubahan yang
diinginkan

git reset:
1. soft: commit tetap ada
2. hard: menghapus commit
yang digunakan adalah soft bukan hard, karena hard akan merusak dari commit jika nanti ada perubahan
